# Package Monitor - 交互设计与 UI 规范文档

## 项目概述

Package Monitor 是一个 macOS 桌面应用，用于监控系统中第三方包的安装情况。基于 Tauri 2 + Vue 3 构建，遵循 Apple Human Interface Guidelines。

---

## 交互设计原则

基于 Interaction Design Skill 的五大维度和 Norman 六原则：

### 1. 系统状态可见性 (Visibility)

| 状态 | 当前实现 | 规范要求 |
|------|----------|----------|
| 扫描中 | 按钮变红 + 旋转动画 + "扫描中..." | ✅ 符合 |
| 空状态 | 虚线圆形 + 提示文字 | ✅ 符合 |
| 加载状态 | 底部状态栏变蓝 | ✅ 符合 |
| 错误状态 | 状态栏显示错误信息 | ⚠️ 需改进 |

**改进项：**
- 错误状态应使用红色背景，与正常状态区分
- 添加 toast 通知组件，提供更明显的错误反馈

### 2. 反馈及时性 (Feedback)

| 操作 | 反馈时间 | 规范 |
|------|----------|------|
| 点击扫描 | 即时变红 | ✅ <100ms |
| 扫描完成 | 状态更新 | ✅ 符合 |
| 悬停按钮 | 颜色变化 | ✅ 符合 |
| 点击日期 | 立即选中 | ✅ 符合 |

**当前问题：**
- 扫描按钮点击后变红，但用户可能不知道为什么变红
- 建议：在变红前先显示"正在启动扫描..."

### 3. 约束与限制 (Constraints)

| 场景 | 实现方式 | 评价 |
|------|----------|------|
| 扫描中禁用按钮 | `:disabled="busy"` | ✅ 正确 |
| 无日期时禁用报告 | `:disabled="!selectedDate"` | ✅ 正确 |
| 定时扫描运行时禁用选择 | `:disabled="schedule.enabled"` | ✅ 正确 |

### 4. 一致性 (Consistency)

**颜色系统：**
- 主色：#007AFF (SF Blue) ✅
- 危险色：#FF3B30 (SF Red) ✅
- 成功色：#34C759 (SF Green) ✅
- 警告色：#FF9500 (SF Orange) ✅

**按钮样式：**
- Primary：渐变蓝色 ✅
- Secondary：灰色边框 ✅
- Danger：渐变红色 ✅
- Small：紧凑尺寸 ✅

### 5. 交互启示 (Affordance)

| 元素 | 启示方式 | 评价 |
|------|----------|------|
| 按钮 | 阴影 + 悬停提升 | ✅ 清晰 |
| 输入框 | 聚焦蓝框 | ✅ 清晰 |
| 日期项 | 悬停背景变化 | ✅ 清晰 |
| 下拉框 | 箭头图标 | ✅ 清晰 |

## UI 设计规范

基于 macOS HIG 和 Super Design Core：

### 1. 颜色系统

```css
:root {
  /* 主色 */
  --sf-blue: #007AFF;
  --sf-blue-light: #409CFF;
  --sf-blue-dark: #0051D5;
  
  /* 功能色 */
  --sf-green: #34C759;
  --sf-red: #FF3B30;
  --sf-orange: #FF9500;
  --sf-purple: #5856D6;
  
  /* 中性色 */
  --sf-text: #1D1D1F;
  --sf-text-sec: #6E6E73;
  --sf-bg: #FFFFFF;
  --sf-sidebar: #F5F5F7;
}
```

### 2. 间距系统

- 页面内边距：20px
- 侧边栏宽度：220px
- 卡片内边距：12-16px
- 元素间距：8-12px
- 按钮内边距：6px 12px

### 3. 字体系统

- 标题：14-15px, font-weight: 600
- 正文：12-13px, font-weight: 400
- 标签：11px, font-weight: 600, uppercase
- 数字：20px, font-weight: 600

### 4. 圆角系统

- 卡片/面板：8-10px
- 按钮：6px
- 输入框：5-6px
- 徽章：4px
- 头像/图标：7px

### 5. 阴影系统

- 卡片阴影：`0 1px 3px rgba(0,0,0,0.04)`
- 悬停阴影：`0 2px 6px rgba(0,122,255,0.4)`
- 按钮阴影：`0 1px 3px rgba(0,122,255,0.3)`

## 按钮状态规范

### 扫描按钮状态机

```
┌─────────┐    点击    ┌─────────┐   完成   ┌─────────┐
│  Ready  │ ────────→ │ Scanning│ ──────→ │  Ready  │
│ (Blue)  │           │  (Red)  │         │ (Blue)  │
└─────────┘           └─────────┘         └─────────┘
      │                     │
      │      错误           │
      │←────────────────────│
      │   (保持红色)        │
```

**状态定义：**

| 状态 | 颜色 | 图标 | 文字 | 光标 | 可点击 |
|------|------|------|------|------|--------|
| Ready | 蓝色渐变 | 时钟 | 立即扫描 | pointer | ✅ |
| Scanning | 红色渐变 | 旋转圈 | 扫描中... | wait | ❌ |
| Error | 红色渐变 | 时钟 | 立即扫描 | pointer | ✅ |

**过渡动画：**
- 颜色过渡：150ms ease
- 图标旋转：800ms linear infinite

## 交互流程规范

### 1. 扫描流程

```
用户点击"立即扫描"
    ↓
按钮立即变红 (0ms)
    ↓
显示旋转动画
    ↓
状态栏显示"扫描中..."
    ↓
后端开始扫描
    ↓
扫描完成
    ↓
按钮恢复蓝色
    ↓
状态栏显示结果
    ↓
更新侧边栏数据
```

**关键时间点：**
- 点击到变红：< 50ms
- 状态栏更新：< 100ms
- 扫描完成到UI更新：< 200ms

### 2. 日期选择流程

```
用户点击日期项
    ↓
立即高亮选中项
    ↓
右侧表格更新
    ↓
状态栏显示日期信息
```

### 3. 定时扫描流程

```
用户选择间隔
    ↓
点击"启动"
    ↓
按钮变红，显示"停止"
    ↓
状态指示灯变绿并脉动
    ↓
状态栏显示"定时扫描已启动"
```

## 错误处理规范

### 1. 扫描错误

**当前实现：** 状态栏显示错误信息
**改进方案：**
- 使用红色背景状态栏
- 添加重试按钮
- 3秒后自动恢复

### 2. 网络错误

**当前实现：** 无特殊处理
**改进方案：**
- 显示 toast 通知
- 提供重试按钮
- 记录错误日志

### 3. 数据加载错误

**当前实现：** 状态栏显示"加载失败"
**改进方案：**
- 显示空状态页面
- 添加重试按钮
- 提供离线提示

## 可访问性规范

### 1. 键盘导航

- Tab 键顺序：扫描按钮 → 报告按钮 → 搜索框 → 日期列表 → 表格
- Enter 键激活按钮
- Escape 键清除搜索

### 2. 屏幕阅读器

- 按钮添加 aria-label
- 状态变化添加 aria-live
- 表格添加 aria-describedby

### 3. 对比度

- 文本与背景对比度 > 4.5:1
- 图标与背景对比度 > 3:1
- 焦点状态有明显边框
