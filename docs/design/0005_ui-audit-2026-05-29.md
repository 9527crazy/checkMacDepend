# UI 审查报告 — 2026-05-29

## 审查范围

对 `src/App.vue` 全部模板、脚本、样式，以及 `src/styles.css` 全局样式进行交叉审查。  
审查依据：Interaction Design Skill（Norman 六原则 + 五维度）、Apple HIG、Super Design Core。

---

## 一、已确认问题清单

### 🔴 P0 — 必须修复

#### 1. `styles.css` 全局样式污染（根因问题）

**现象**：`src/styles.css` 在 `main.js` 中被全局导入，包含一整套旧的 teal/green 配色方案（`#0f766e`、`#dff4f0`、`#eef1f5`），与 `App.vue` 中的 SF 蓝配色系统直接冲突。

**具体冲突点**：

| 选择器 | `styles.css` 值 | `App.vue` 值 | 后果 |
|--------|-----------------|-------------|------|
| `body` | `background: #eef1f5` | `background: var(--sf-gray-6)` (`#F2F2F7`) | 背景色不一致 |
| `.app-shell` | `padding: 24px` | `height: 100vh`，无 padding | 布局被撑开 |
| `.workspace` | `display: grid; grid-template-columns: 220px 1fr` | `display: flex` | 布局模型冲突 |
| `.date-item.active` | `background: #dff4f0; color: #0f766e` | `background: var(--sf-blue); color: white` | **日期选中颜色被覆盖** |
| `.date-item:hover` | `background: #dff4f0; color: #0f766e` | `background: var(--sf-gray-5)` | hover 颜色异常 |
| `.th`, `.td` | `padding: 12px 14px; font-size: 14px` | `padding: 10px 12px; font-size: 11-12px` | 表格行高不一致 |
| `.empty-state` | `color: #64748b; padding: 40px; text-align: center` | `flex: 1; display: flex; flex-direction: column` | 空状态布局冲突 |

**影响**：这是日期颜色看不清、表格高度不一致、整体视觉混乱的根本原因。

**修复**：删除 `src/styles.css`，或将其中仍需保留的基础重置（`box-sizing`、`margin: 0`）合并到 `App.vue` 的 `<style>` 中。

---

#### 2. 日期文字不可读

**现象**：选中日期后，文字颜色不正确，与背景色对比度极低。

**根因**：`styles.css` 中 `.date-item.active` 设置 `color: #0f766e`（深青色），与 `App.vue` 中 `color: white` 冲突。由于 CSS 后定义优先（同 specificity），但 `styles.css` 的 `:hover` 规则也在干扰。

**具体表现**：
- 选中日期背景：`#007AFF`（蓝色）✅
- 选中日期文字：被 `styles.css` 覆盖为 `#0f766e`（青绿色）❌
- 对比度：`#0f766e` on `#007AFF` → 对比度约 2.1:1，远低于 WCAG AA 4.5:1

---

#### 3. 表格高度比侧边栏矮一截

**现象**：右侧表格内容区域比左侧侧边栏矮，底部有空白。

**根因**：
- `styles.css` 的 `.workspace` 使用 `grid-template-columns: 220px 1fr`，而 `App.vue` 使用 `display: flex`
- `styles.css` 的 `.table-panel` 有 `min-height: calc(100vh - 188px)`，而 `App.vue` 的 `.content` 使用 `flex: 1`
- 两种布局模型叠加导致高度计算不一致

### 🟡 P1 — 应该修复

#### 4. 侧边栏文字过小

**当前字号统计**：

| 元素 | 当前字号 | HIG 建议最小值 | 问题 |
|------|---------|---------------|------|
| `.sidebar-title` | 10px | 11px | ❌ 过小 |
| `.stat-label` | 10px | 11px | ❌ 过小 |
| `.stat-key` | 11px | 12px | ⚠️ 偏小 |
| `.stat-val` | 11px | 12px | ⚠️ 偏小 |
| `.schedule-status` | 11px | 12px | ⚠️ 偏小 |
| `.date-count` | 10px | 11px | ❌ 过小 |
| `.history-item` | 12px | 12px | ✅ 正常 |

**macOS HIG 参考**：
- 辅助文字最小字号：11px（SF Pro Text Caption）
- 正文最小字号：13px（SF Pro Text Body）
- 侧边栏标签：12-13px

---

#### 5. 侧边栏宽度不足导致文字截断

**现象**：`.stat-val` 设置了 `max-width: 100px`，在 200px 宽的侧边栏中，管理器名称（如 "homebrew, pip, npm"）会被截断显示为省略号。

**当前侧边栏结构**：
```
.sidebar { width: 200px }
  .sidebar-section { padding: 12px }
    .stat-val { max-width: 100px }  ← 可用宽度仅约 176px
```

**影响**：用户无法看到完整的管理器列表，需要 hover 才能看到 `title` 属性。

---

#### 6. 表格列宽分配不合理

**现象**：表格四列（管理器、包名、版本、时间）没有明确的列宽分配，导致：
- 管理器列（badge）占过多空间
- 包名列可能被挤压
- 版本列（monospace）换行或过窄

**当前 CSS**：无 `th`/`td` 的 `width` 或 `min-width` 定义。

---

### 🟢 P2 — 可以优化

#### 7. 骨架屏使用 `Math.random()` 导致不稳定

**现象**：骨架屏宽度使用 `:style="{ width: (40 + Math.random() * 30) + '%' }"`，每次渲染随机宽度，但 Vue 的响应式系统会在每次 re-render 时重新计算，导致骨架屏宽度不断跳动。

**修复**：使用固定宽度数组或 CSS `:nth-child` 伪类。

---

#### 8. 扫描历史缺少视觉层次

**现象**：历史记录项使用绿色圆点 + 时间 + 计数，但所有项视觉权重相同，无法快速区分最近一次扫描。

**建议**：最近一次扫描加粗或使用不同颜色圆点。

---

#### 9. Toast 通知无手动关闭

**现象**：Toast 3 秒后自动消失，但用户无法手动关闭。如果同时出现多条 toast，可能遮挡右上角内容。

**建议**：添加关闭按钮或支持点击消失。

---

#### 10. 状态栏 `role="status"` + `aria-live="polite"` 可能过度通知

**现象**：状态栏每次文字变化都会触发屏幕阅读器通知（如从 "就绪" → "扫描中..." → "扫描中... 已发现 1 个包" → ...），可能导致频繁通知干扰。

**建议**：仅在操作完成（成功/失败）时使用 `aria-live`，扫描过程中使用 `aria-live="off"`。

## 四、总结

**核心问题**：`src/styles.css` 的全局样式与 `App.vue` 的内联样式存在系统性冲突，是日期颜色、表格高度、视觉混乱三大问题的根因。删除或重构此文件即可一次性解决 P0 级问题。

**建议下一步**：
1. 删除 `src/styles.css`，将必要的基础重置（`box-sizing`、`margin: 0`、`body` 字体）移入 `App.vue`
2. 修复侧边栏字号和宽度
3. 重新审查修复后的界面
