## 背景

扫描是耗时操作（几秒到十几秒），当前用户只能看到旋转的"扫描中..."按钮，无法了解具体进展。需要一个实时日志面板，展示扫描的详细进度和执行步骤。

## 目标与范围

- 提供实时日志面板，展示扫描过程中的每个步骤
- 以可折叠面板形式展示，默认折叠，扫描时自动展开
- 支持自动滚动、清空日志、复制日志

## 设计方案

### 面板位置

- 主内容区底部，可折叠面板
- 默认折叠，扫描时自动展开
- 可拖拽调整高度

### 日志格式

每条日志包含：

- **时间戳**：`HH:MM:SS` 格式
- **级别**：`info` / `success` / `warning` / `error`（不同颜色）
- **消息**：具体描述

示例：

```
11:23:01  [info]     开始扫描系统包...
11:23:01  [info]     检测 Homebrew...
11:23:02  [info]     Homebrew: 找到 142 个包
11:23:02  [info]     检测 pip3...
11:23:03  [info]     pip3: 找到 89 个包
11:23:03  [info]     检测 npm...
11:23:04  [info]     npm: 找到 56 个包
11:23:04  [info]     检测 cargo...
11:23:05  [info]     cargo: 找到 23 个包
11:23:05  [success]  扫描完成！共 310 个包，发现 5 个新增包
```

### 后端实现

使用 Tauri 事件系统：

```rust
#[derive(Clone, Serialize)]
struct ScanLogPayload {
    timestamp: String,  // HH:MM:SS
    level: String,      // info / success / warning / error
    message: String,
}
```

将 `scan_all_packages()` 拆分为逐个管理器扫描，每个完成后发送进度事件。

### 前端实现

- 新增日志面板 UI
- 监听 `scan-log` 事件，追加日志条目
- 支持自动滚动到底部
- 提供"清空"和"复制日志"按钮

### UI 交互

1. 点击"立即扫描"后自动展开日志面板
2. 扫描完成后日志保留，可手动折叠
3. 每次扫描开始时自动清空之前的日志
