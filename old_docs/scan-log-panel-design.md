# 扫描日志面板（Execution Log Panel）

## 概述

在扫描过程中，为用户提供一个实时日志面板，展示当前扫描的详细进度和执行步骤。类似终端输出，让用户清晰了解正在发生什么。

## 当前问题

扫描是一个耗时操作（可能需要几秒到十几秒），但目前用户只能看到一个旋转的"扫描中..."按钮，不知道具体进展。

## 设计方案

### 面板位置

- 在主内容区底部，以可折叠面板形式展示
- 默认折叠，扫描时自动展开
- 可拖拽调整高度

### 日志内容

每条日志包含：
- **时间戳**：HH:MM:SS 格式
- **级别**：info / success / warning / error（不同颜色）
- **消息**：具体描述

示例日志流：
```
11:23:01  [info]  开始扫描系统包...
11:23:01  [info]  检测 Homebrew...
11:23:02  [info]  Homebrew: 找到 142 个包
11:23:02  [info]  检测 pip3...
11:23:03  [info]  pip3: 找到 89 个包
11:23:03  [info]  检测 npm...
11:23:04  [info]  npm: 找到 56 个包
11:23:04  [info]  检测 cargo...
11:23:05  [info]  cargo: 找到 23 个包
11:23:05  [success] 扫描完成！共 310 个包，发现 5 个新增包
```

### 实现方式

**后端 (Rust)：**
- 使用 Tauri 事件系统（`app.emit()`）在扫描过程中发送日志事件
- 将 `scan_all_packages()` 拆分为逐个管理器扫描，每个完成后发送进度事件

**前端 (Vue)：**
- 新增日志面板 UI
- 监听 `scan-log` 事件，追加日志条目
- 支持自动滚动到底部
- 清空日志按钮

### 事件定义

```rust
#[derive(Clone, Serialize)]
struct ScanLogPayload {
    timestamp: String,  // HH:MM:SS
    level: String,      // info / success / warning / error
    message: String,
}
```

### UI 交互

1. 点击"立即扫描"后自动展开日志面板
2. 扫描完成后日志保留，可手动折叠
3. 提供"清空"按钮清除日志
4. 提供"复制日志"按钮

### 文件变更

| 文件 | 变更 |
|------|------|
| `src-tauri/src/main.rs` | 拆分扫描逻辑，逐个管理器发送事件 |
| `src/App.vue` | 新增日志面板 UI + 事件监听 |

### 日志行为

- 每次扫描开始时自动清空之前的日志
- 扫描过程中实时追加日志
- 扫描完成后日志保留，用户可手动清空
